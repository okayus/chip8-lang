# chip8-lang: CHIP-8 向け関数型言語

CHIP-8 ROM にコンパイルできる小さな関数型言語を Rust で実装する。
学習目的のため厳密さよりも「言語処理系の各フェーズを一通り体験する」ことを優先する。

## ゴール

- TETRIS が書ける程度の機能を持つ関数型言語を設計・実装する
- Lexer → Parser → AST → コード生成 → `.ch8` バイナリ出力 のパイプラインを一通り作る
- 言語処理系の仕組みを学ぶ

## 言語の概要

### 設計方針

- ML 系のシンタックス (let 束縛、パターンマッチ、関数定義)
- 静的型付け (型推論なし・明示的な型注釈)
- CHIP-8 のハードウェア制約に合わせたプリミティブ型
- 組み込み関数で CHIP-8 固有の操作 (描画・入力・タイマー) を提供

### 型システム

```
u8          -- 8ビット符号なし整数 (CHIP-8 レジスタに対応)
bool        -- 真偽値 (内部的には u8)
[u8; N]     -- 固定長配列 (メモリ上に配置)
sprite(N)   -- N バイトのスプライトデータ (リテラルとして定義)
```

### 構文イメージ

```
-- 定数定義
let BOARD_W: u8 = 10;
let BOARD_H: u8 = 20;

-- スプライト定義 (バイナリリテラル)
let block_sprite: sprite(1) = [0b11000000];

-- 関数定義
fn draw_block(x: u8, y: u8) -> () {
  draw(block_sprite, x, y);
}

-- 条件分岐 (if 式)
fn clamp(val: u8, max: u8) -> u8 {
  if val > max { max } else { val }
}

-- ループ (loop + break)
fn game_loop() -> () {
  loop {
    let key = wait_key();
    if key == 5 {
      break;
    };
    -- ...
  }
}

-- エントリーポイント
fn main() -> () {
  clear();
  game_loop();
}
```

### 組み込み関数 (CHIP-8 命令へのマッピング)

| 関数                        | CHIP-8 命令     | 説明                         |
|----------------------------|-----------------|------------------------------|
| `clear()`                  | `00E0`          | 画面クリア                   |
| `draw(sprite, x, y) -> bool` | `DXYN`       | スプライト描画、衝突を返す   |
| `wait_key() -> u8`         | `FX0A`          | キー入力待ち                 |
| `is_key_pressed(k) -> bool`| `EX9E`          | キーが押されているか         |
| `delay() -> u8`            | `FX07`          | ディレイタイマー読み取り     |
| `set_delay(v: u8)`         | `FX15`          | ディレイタイマー設定         |
| `set_sound(v: u8)`         | `FX18`          | サウンドタイマー設定         |
| `random(mask: u8) -> u8`   | `CXKK`          | 乱数生成 (AND mask)          |
| `bcd(v: u8)`               | `FX33`          | BCD 変換                     |
| `draw_digit(v: u8, x, y)`  | `FX29` + `DXYN` | 16進数字スプライト描画       |

## コンパイラのアーキテクチャ

```
ソースコード (.ch8l)
    │
    ▼
┌──────────┐
│  Lexer   │  ソース文字列 → トークン列
└──────────┘
    │
    ▼
┌──────────┐
│  Parser  │  トークン列 → AST
└──────────┘
    │
    ▼
┌──────────┐
│ Analyzer │  型チェック・変数解決・レジスタ割り当て
└──────────┘
    │
    ▼
┌──────────┐
│ CodeGen  │  AST → CHIP-8 バイトコード
└──────────┘
    │
    ▼
┌──────────┐
│ Emitter  │  バイトコード → .ch8 バイナリファイル
└──────────┘
```

## 実装フェーズ

### Phase 1: Lexer
トークン定義と字句解析。

- トークン型: キーワード (`let`, `fn`, `if`, `else`, `loop`, `break`, `return`, `true`, `false`)、
  識別子、数値リテラル (10進, 16進 `0x`, 2進 `0b`)、演算子、区切り文字
- 入力文字列を走査してトークン列を生成
- 行番号・列番号をトークンに付与 (エラー報告用)

### Phase 2: Parser
再帰下降パーサーで AST を構築。

- AST ノード型の定義 (enum で代数的データ型として表現)
- 式: リテラル、変数参照、関数呼び出し、二項演算、if 式
- 文: let 束縛、代入、式文、return、break
- トップレベル: 関数定義 (`fn`)、定数定義 (`let`)、スプライト定義
- 演算子の優先順位処理 (Pratt parsing)

### Phase 3: 意味解析
型チェックとシンボル解決。

- スコープ管理 (グローバル / 関数ローカル)
- 型チェック (型注釈ベース、推論なし)
- 未定義変数・関数の検出
- CHIP-8 制約の検証 (ローカル変数は最大16個 = レジスタ数)

### Phase 4: コード生成
AST から CHIP-8 バイトコードを生成。

- レジスタ割り当て (V0-VE をローカル変数に対応、VF はフラグ専用)
- 関数呼び出し → `CALL` / `RET` 命令
- 条件分岐 → skip 命令 (`SE` / `SNE`) + `JP` の組み合わせ
- ループ → ラベル管理 + `JP` でループバック
- スプライトデータ → ROM 末尾にデータセクションとして配置
- 配列アクセス → `I` レジスタ + `LD`/`ST` 命令

### Phase 5: バイナリ出力 & 統合テスト
`.ch8` ファイルを出力し、既存の chip8-ts エミュレータで動作確認。

- バイトコードをバイナリファイルとして書き出し
- テスト ROM を作成して chip8-ts で実行
- TETRIS の実装に挑戦

## CHIP-8 ハードウェア制約メモ

コード生成時に考慮すべき制約:

| リソース           | 制限                                    |
|--------------------|-----------------------------------------|
| メモリ             | 4KB (0x000-0xFFF)、プログラムは 0x200 開始 |
| 汎用レジスタ       | V0-VF の 16 本 (各 8bit)、VF はフラグ   |
| スタック           | 16 段 (サブルーチンのネスト上限)        |
| 画面               | 64×32 ピクセル (モノクロ)               |
| スプライト         | 幅 8px 固定、高さ 1-15px               |
| タイマー           | DT, ST (60Hz でデクリメント)            |
| キー入力           | 16 キー (0x0-0xF)                       |

## 技術スタック

- **言語**: Rust (2024 edition)
- **テスト**: 標準の `#[test]`
- **依存**: 最小限 (基本的に std のみ)
- **検証**: chip8-ts エミュレータで動作確認
