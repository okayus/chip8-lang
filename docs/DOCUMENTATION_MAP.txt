【実装学習コース】モジュール毎に詳しく

  + tests/{*_tests.rs} を読む (1時間)
  + cargo test でテスト実行 (30分)
  ────────────────────────────────
  合計: 4時間15分


【完全習得コース】深い理解+カスタマイズ
  QUICK_START.md (15分)

  + DESIGN.md (30分)
  + IMPLEMENTATION_GUIDE.md (2時間)
  + ソースコード詳細読み込み (2時間)
  + テストコード学習 (1.5時間)
  + 自分で機能拡張・実装 (2時間)
  ────────────────────────────────
  合計: 9時間

QUICK_START.md (高速スタート)
  ├─→ DESIGN.md (言語仕様・制約確認)
  └─→ IMPLEMENTATION_GUIDE.md (詳細学習)


HOW_IT_WAS_BUILT.md (実装背景)
  ├─ 実装方針の解説
  ├─ DESIGN.md への参照

  ├─ QUICK_START で見た概要を詳しく説明

  ├─ 実装例（hello.ch8l など）
  └─→ src/ と tests/ のコード読み込みへ


┌─ src/ ソースコード

cargo build

Q1: どのドキュメントから始めるべき？
A1: README.md から始めてください。その後、時間があれば QUICK_START.md。
    これだけで全体像が把握できます。

Q2: 実装の詳細を知りたい場合は？
A2: IMPLEMENTATION_GUIDE.md がメイン教材です。
    QUICK_START.md で概要をつかんでから読むことをお勧めします。

Q3: 実装哲学を理解したい場合は？
A3: HOW_IT_WAS_BUILT.md を読んでください。
    「型でドメインを表現する」という中心的な考え方が理解できます。

Q4: ソースコードはいつ読むべき？
A4: IMPLEMENTATION_GUIDE.md の該当セクションで
    「ファイル参照」が示されている箇所を、そのタイミングで読むのが効率的です。

Q5: テストはなぜ重要？
A5: テストが言語規則の明示になっているので、
    テストを読むことで仕様が正確に理解できます。

Q6: 拡張機能を追加したい場合は？
A6: 完全習得コース (9時間) を修了した後に、
    テストを書いてから実装を追加してください。

Q7: すべてを読む必要はある？
A7: いいえ。時間に応じて「高速コース」「実装学習コース」「完全習得コース」
    から選んでください。
